**Step 3 — Override CSS variables in src/index.css:**
After shadcn init creates the CSS variable block, replace the `:root` and `.dark` blocks with the exact values from the UI-SPEC color palette. The final `src/index.css` must contain these exact custom properties (using oklch or HSL as shadcn generates, but the teal values must resolve to these hex equivalents):

```css
/* Add at top of src/index.css, before or after existing @tailwind directives */
@import '@fontsource/inter/400.css';
@import '@fontsource/inter/600.css';

/* Override shadcn CSS variables to match Conecta Bairro palette */
:root {
  --background: #FAFAFA;
  --foreground: #0F172A;
  --card: #F0FDFA;
  --card-foreground: #0F172A;
  --primary: #0D9488;
  --primary-foreground: #FFFFFF;
  --destructive: #DC2626;
  --destructive-foreground: #FFFFFF;
  --border: #E2E8F0;
  --input: #E2E8F0;
  --ring: #0D9488;
  --muted: #F1F5F9;
  --muted-foreground: #64748B;
  --secondary: #F0FDFA;
  --secondary-foreground: #0F172A;
  --accent: #F0FDFA;
  --accent-foreground: #0D9488;
  --popover: #FAFAFA;
  --popover-foreground: #0F172A;
  --radius: 0.75rem;
}
```

IMPORTANT: shadcn may use oklch() color format in newer versions. If it does, convert the hex values to oklch equivalents using the correct conversion:
- `#FAFAFA` → `oklch(0.985 0 0)` (near-white)
- `#0D9488` → `oklch(0.591 0.148 187.6)` (teal-600)
- `#0F172A` → `oklch(0.129 0.042 264.7)` (slate-900)
- `#DC2626` → `oklch(0.577 0.245 27.3)` (red-600)
- `#F0FDFA` → `oklch(0.984 0.014 185.7)` (teal-50)

Use whichever format shadcn is already using in the generated CSS to stay consistent. Do NOT mix hex and oklch.
